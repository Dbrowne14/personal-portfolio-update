DAVID BROWNE — PORTFOLIO · DESIGN SCHEMA · V2 — LIGHT-FIRST

Signal, not noise.

One system, four pages, one story: a career that moved from pricing technology to building it. Every decision below serves that narrative.

01 — DESIGN PHILOSOPHY

Editorial precision with one moment of ambition per page. The site behaves like a well-typeset financial document that has learned to code: hairline rules, monospace metadata, restrained color — punctuated by exactly one signature interaction per page. Interaction is always informational (scrubbing data, revealing structure), never decorative. If an animation doesn't carry meaning, it doesn't ship.

02 — GRID & LAYOUT
  • Full-bleed canvas with a 28px page gutter; content never boxed in a centered column on the site itself (this document is the exception).
  • Structure is expressed by 1px hairlines (var --line) and occasional full-bleed ink/bronze plates — never by grey blocks, cards, shadows, or rounded containers.
  • Ruled grids: repeat(auto-fit, minmax(270–320px, 1fr)) — cells divided by hairlines, collapsing gracefully to a single ruled column on mobile.
  • Section headers pair a display headline (left) with a mono section index "0N — LABEL" (right), giving every page the same scanning rhythm.

Designed light-first. The defining surface is warm bond ivory with a soft warm ink — a printed page, not a screen — and one accent: burnished bronze, the pigment of ledger annotations, deep enough to pass AA even at mono caption sizes. The canvas stays predominantly white; separation comes from hairlines and full-bleed plates — an ink plate behind the career chart and toolset band — plus warm bronze washes (7–9% accent tints), never grey blocks. Elsewhere color marks what is active, current, or clickable-forward. Derived tones only: ink at 72% / 50%, hairlines at 15–16%. Dark mode is the same page under different lighting: identical warm hue family (never blue-black), the bronze lifted just enough to hold contrast, swapped via CSS custom properties — no second palette exists. Brand-adjacent pairings (green on near-black, red on pink) were explicitly rejected. No gradients, no glass, no purple.

05 — SPACING & SIZING
  • Base-4 scale; workhorses: 12 / 20 / 28 (page gutter) / 56 (section padding) / 90–130 (hero breathing room).
  • Heroes claim 100vh; content sections size to content — no filler height.
  • Interactive rows ≥ 44px tall everywhere; mobile timeline rows 76px.
  • Radius: 0. The only rounded element is the theme-toggle pill — a deliberate quotation from the old site.

07 — MOTION PRINCIPLES
  • One signature interaction per page: monument (home), chart scrub (journey), halftone portrait (about), row expansion (work). Nothing else animates on its own.
  • Micro-interactions are limited to: grayscale→color, hairline/bg lift, border inversion. 200–250ms, ease-out.
  • Motion is input-driven (cursor, scroll, tap) — the site is still when the visitor is still.
  • Physics feel: lerp factor ~0.12 for magnetics, damped springs for particles; entrance choreography ≤ 900ms total, once.
  • Performance budget: transforms + canvas only; DPR capped at 2; rAF loops cancelled on unmount; no layout-triggering animation.
08 — RESPONSIVE BEHAVIOR
  • Type scales fluidly via clamp(); grids collapse via auto-fit; nothing is merely shrunk.
  • Navigation / mobile: masthead reduces to name + burger; the menu is a full-screen ledger takeover (see §06), not a drawer.
  • Home / mobile: cursor magnetics off; the monument keeps its entrance assemble and the scroll-compress into the masthead — scroll is the mobile input.
  • Journey / mobile: the chart rotates 90° into a vertical timeline along the scroll axis; milestones become full-width tap targets.
  • About / mobile: halftone portrait renders as a static print-halftone artwork (dot size maps to tonal darkness in light mode, luminance in dark — always a positive image, never a negative) below the intro (coarse pointers get no scatter loop).
  • Work / mobile: ruled column; case-study grid stacks text-first, image second.

10 — SIGNATURE INTERACTIONS
  • The Monument (home): magnetic letters + scroll-compress into masthead. Entrance assemble on load.
  • The Chart (home §2): scrubbable career timeline; curve→ticks quantization at the 2025 switch. The site's conceptual heart.
  • The Halftone (about): portrait as particle field, scattering and reforming around the cursor.
  • The Ledger (work): index rows expanding in place — navigation as typography.
11 — ACCESSIBILITY & REDUCED MOTION
  • prefers-reduced-motion: all rAF loops skipped (canvases render one static frame), entrances and transitions collapse to instant states, scroll-compress becomes a threshold swap. Scrubbing remains — it's user-initiated data inspection, not motion.
  • Text never sits inside canvases — charts and particles are decorative layers behind real HTML; milestone data also exists as DOM text (mobile timeline / experience list).
  • Contrast: warm ink on ivory ≈ 15:1; ink-70 ≥ 8:1; bronze accent ≥ 5.5:1 on ivory; accent is never the only signal (paired with +/×, underline, or label).
  • Keyboard: rows are focusable with visible accent focus baselines; nav is plain links; hit targets ≥ 44px.
